# Run-Entry Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Assemble the engine/room/gateway libraries into a runnable `npm start` process that can be exercised end-to-end with wscat.

**Architecture:** A pure `loadConfig(env)` reads runtime params from environment with defaults. A pure `createServer(config, opts)` composition layer breaks the gateway↔manager circular dependency (build gateway → use it as the RoomManager's transport → `attachRoomManager` back into the gateway), starts the ws server via an injectable factory (real `WebSocketServer` by default, fake in tests), and returns an idempotent `shutdown()`. A thin `main.ts` wires `process.env` and SIGINT/SIGTERM signals. Gateway gains one method, `stopHeartbeat()`, used only for graceful shutdown.

**Tech Stack:** TypeScript (ESM, `verbatimModuleSyntax` → use `import type` for types), Vitest, `ws`, `tsx` (new dev dependency, run entry).

## Environment / How to Run Commands

The sandbox segfaults (exit 139) on bare `npx`/`npm`. Always run from `server/` with bundled CA, using local binaries:

- Tests (single file): `cd server && NODE_OPTIONS="--use-bundled-ca" ./node_modules/.bin/vitest run src/<path>.test.ts`
- All tests: `cd server && NODE_OPTIONS="--use-bundled-ca" ./node_modules/.bin/vitest run`
- Typecheck: `cd server && ./node_modules/.bin/tsc --noEmit`

`tsx` must be installed before `npm start` works. Installing deps in the sandbox needs the special flags from project memory (`--use-bundled-ca`, cleared proxy env, fresh `--cache`, `--registry=http://registry.m.jd.com`, `dangerouslyDisableSandbox:true`). If `tsx` can't be installed in-sandbox, add it to `package.json` and let the user run install on their machine — the unit tests in this plan do NOT depend on `tsx`.

`main.ts` is NOT unit-tested (touches real `process`/signals; sandbox can't run a long-lived process). It is verified by the user via wscat on their machine.

```typescript
// server/src/config.ts
export interface WxConfig {
  appid: string
  secret: string
}

export interface Config {
  port: number
  heartbeatMs: number
  turnMs: number
  capacity: number
  wx: WxConfig | null
}

function readPositiveInt(env: NodeJS.ProcessEnv, key: string, fallback: number): number {
  const raw = env[key]
  if (raw === undefined || raw === '') return fallback
  const n = Number(raw)
  if (!Number.isInteger(n) || n <= 0) throw new Error(`invalid ${key}: ${raw}`)
  return n
}

export function loadConfig(env: NodeJS.ProcessEnv): Config {
  const appid = env.WX_APPID
  const secret = env.WX_SECRET
  return {
    port: readPositiveInt(env, 'PORT', 8080),
    heartbeatMs: readPositiveInt(env, 'HEARTBEAT_MS', 30000),
    turnMs: readPositiveInt(env, 'TURN_MS', 30000),
    capacity: readPositiveInt(env, 'ROOM_CAPACITY', 3),
    wx: appid && secret ? { appid, secret } : null,
  }
}
```

## Task 3: Composition layer (`createServer`)

**Files:**
- Create: `server/src/composition.ts`
- Test: `server/src/composition.test.ts`

This is the core task. `createServer` must: select the authenticator from config, build the gateway, build the RoomManager using the gateway as transport, **call `gateway.attachRoomManager(manager)`** (the step `createGateway` currently omits), start the ws server via an injectable factory, start the heartbeat, and return an idempotent `shutdown()`.

The injectable `WssFactory` lets tests avoid binding a real port and lets us capture the `connection` handler. `WssLike` is the minimal surface `createServer` needs from a ws server.

- [ ] **Step 1: Write the failing tests**

```typescript
// server/src/composition.test.ts
import { describe, it, expect } from 'vitest'
import { createServer, type WssLike } from './composition'
import type { Config } from './config'
import { WechatAuthenticator, StubAuthenticator, FakeSocket } from './gateway'

// A fake ws server: records the connection handler, exposes close() call count.
function fakeWssFactory() {
  const calls = { close: 0 }
  let onConnection: ((ws: unknown) => void) | undefined
  const factory = (_port: number): WssLike => ({
    on: (event, handler) => { if (event === 'connection') onConnection = handler as (ws: unknown) => void },
    close: () => { calls.close++ },
    clients: new Set(),
  })
  return { factory, calls, getOnConnection: () => onConnection }
}

const baseConfig: Config = { port: 8080, heartbeatMs: 1000, turnMs: 1000, capacity: 3, wx: null }

describe('createServer', () => {
  it('selects StubAuthenticator when wx is null', () => {
    const { factory } = fakeWssFactory()
    const server = createServer(baseConfig, { wssFactory: factory })
    expect(server.authenticator).toBeInstanceOf(StubAuthenticator)
  })

  it('selects WechatAuthenticator when wx is set', () => {
    const { factory } = fakeWssFactory()
    const server = createServer(
      { ...baseConfig, wx: { appid: 'a', secret: 's' } },
      { wssFactory: factory },
    )
    expect(server.authenticator).toBeInstanceOf(WechatAuthenticator)
  })

  it('attaches the RoomManager to the gateway (no this.manager! crash on JOIN)', async () => {
    const { factory } = fakeWssFactory()
    const server = createServer(baseConfig, { wssFactory: factory })
    // Drive AUTH then CREATE through the gateway using the same FakeSocket
    // injection helper gateway.test.ts uses. If the manager were not attached,
    // route() would throw on `this.manager!`. We assert the room got created.
    const s = new FakeSocket()
    server.gateway.handleConnection(s)
    s.receive(JSON.stringify({ type: 'AUTH', payload: { code: 'p1' } }))
    s.receive(JSON.stringify({ type: 'CREATE', payload: { roomId: 'r1' } }))
    await server.gateway.idle()
    expect(server.manager.getRoom('r1')).toBeDefined()
  })

  it('shutdown stops heartbeat, closes wss, and is idempotent', async () => {
    const { factory, calls } = fakeWssFactory()
    const server = createServer(baseConfig, { wssFactory: factory })
    await server.shutdown()
    await server.shutdown()
    expect(calls.close).toBe(1) // second shutdown is a no-op
  })
})
```

NOTE for the implementer: messages go through `FakeSocket.receive(json)` and are wrapped in a `payload` envelope — `{ type:'AUTH', payload:{ code:'p1' } }`, `{ type:'CREATE', payload:{ roomId:'r1' } }` — matching `gateway.test.ts`. `FakeSocket` is re-exported from `./gateway`. Do not assert against private fields.

```typescript
// server/src/composition.ts
import type { Socket } from './gateway/socket'
import type { Config } from './config'
import type { Authenticator } from './gateway'
import type { RoomManager as RoomManagerType } from './room'
import type { WsGateway as WsGatewayType } from './gateway'
import { WsGateway, StubAuthenticator, WechatAuthenticator } from './gateway'
import { RoomManager, InMemorySnapshotStore, realScheduler } from './room'
import { PaodekuaiEngine } from './engine/paodekuai'

// Minimal ws-server surface createServer depends on. Real WebSocketServer
// satisfies this; tests inject a fake to avoid binding a port.
export interface WssLike {
  on(event: 'connection', handler: (ws: unknown) => void): void
  close(): void
  clients: Set<{ close(): void }>
}

export type WssFactory = (port: number) => WssLike

export interface RunningServer {
  gateway: WsGatewayType
  manager: RoomManagerType
  authenticator: Authenticator
  wss: WssLike
  shutdown: () => Promise<void>
}

const intervalScheduler = {
  set: (cb: () => void, ms: number) => setInterval(cb, ms) as unknown as number,
  clear: (id: number) => clearInterval(id),
}

// Wrap a real ws.WebSocket into the gateway's Socket abstraction.
// Mirrors the wrap() in gateway/index.ts; kept here so composition owns
// the lifecycle wiring. Typed loosely because tests inject fakes.
function wrap(ws: any): Socket {
  return {
    send: (data) => ws.send(data),
    close: () => ws.close(),
    terminate: () => ws.terminate(),
    ping: () => ws.ping(),
    on: (event: 'message' | 'close' | 'pong', handler: (...a: never[]) => void) => {
      if (event === 'message') ws.on('message', (d: { toString(): string }) => (handler as (s: string) => void)(d.toString()))
      else if (event === 'close') ws.on('close', () => (handler as () => void)())
      else ws.on('pong', () => (handler as () => void)())
    },
  }
}

function defaultWssFactory(port: number): WssLike {
  // Lazy import keeps the ws dependency confined; real server binds the port.
  const { WebSocketServer } = require('ws')
  return new WebSocketServer({ port }) as unknown as WssLike
}

export function createServer(
  config: Config,
  opts?: { wssFactory?: WssFactory },
): RunningServer {
  const authenticator: Authenticator = config.wx
    ? new WechatAuthenticator(config.wx, globalThis.fetch as never)
    : new StubAuthenticator()

  const gateway = new WsGateway({ authenticator })

  const manager = new RoomManager({
    engine: new PaodekuaiEngine(),
    transport: gateway,
    store: new InMemorySnapshotStore(),
    scheduler: realScheduler,
    capacity: config.capacity,
    turnMs: config.turnMs,
    rngFor: () => Math.random,
  })

  gateway.attachRoomManager(manager)

  const factory = opts?.wssFactory ?? defaultWssFactory
  const wss = factory(config.port)
  wss.on('connection', (ws) => gateway.handleConnection(wrap(ws)))

  gateway.startHeartbeat(intervalScheduler, config.heartbeatMs)

  let stopped = false
  const shutdown = async (): Promise<void> => {
    if (stopped) return
    stopped = true
    gateway.stopHeartbeat()
    wss.close()
    for (const client of wss.clients) client.close()
  }

  return { gateway, manager, authenticator, wss, shutdown }
}
```

NOTE on `require('ws')`: this file is ESM (`"type":"module"`). If `require` is unavailable, switch `defaultWssFactory` to a top-level `import { WebSocketServer } from 'ws'` and construct directly — the lazy form is only to keep the import out of the test path, but tests always inject `wssFactory`, so a top-level import is acceptable too. Use whichever typechecks cleanly under `verbatimModuleSyntax`. Either way, `composition.ts` importing `ws` is fine; the architectural rule is that the gateway is the only layer that *handles* ws connections, and composition merely starts the server.

```typescript
import { loadConfig } from './config'
import { createServer } from './composition'

const config = loadConfig(process.env)
const server = createServer(config)

const authMode = config.wx ? 'wechat' : 'stub'
console.log(`[run-entry] listening on :${config.port} (auth=${authMode}, capacity=${config.capacity})`)

let shuttingDown = false
async function onSignal(sig: string): Promise<void> {
  if (shuttingDown) return
  shuttingDown = true
  console.log(`[run-entry] received ${sig}, shutting down`)
  await server.shutdown()
  console.log('[run-entry] shutdown complete')
  process.exit(0)
}

process.on('SIGINT', () => void onSignal('SIGINT'))
process.on('SIGTERM', () => void onSignal('SIGTERM'))
```
