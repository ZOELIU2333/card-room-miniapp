# 棋牌好友房小程序 PRD

更新时间：2026-06-23

## 1. 项目概述

项目目标是做一个以好友同玩为核心的棋牌微信小程序。用户可以创建好友房、邀请微信好友、输入房号加入牌局，也可以在后续版本进入自由匹配、积分排名和战绩体系。

第一版先不考虑充值系统。积分只作为游戏内成长、战绩和轻竞技反馈，不与充值、提现或现金兑换绑定。

## 2. 产品定位

不是传统棋牌大厅，也不是强竞技手游大厅，而是一个高质感的好友约局工具。

关键词：

- 好友房优先
- 微信小程序优先
- 简约高级
- 深色调
- 低动画
- 快速开局
- 清晰可用
- 积分反馈

## 3. 目标用户

主要用户是平时喜欢和熟人打牌、打麻将的人，偏好快速约局、建房分享和稳定开局。

典型场景：

- 几个朋友晚上临时约一桌
- 固定牌友群周末开局
- 熟人之间玩积分，不涉及充值
- 后续可通过自由匹配补足空位或拓展陌生人玩法

## 4. 支持玩法范围

用户明确提到的棋牌类型：

- 跑得快，支持 15 张、16 张
- 三打哈
- 240
- 字牌
- 掼蛋
- 德州扑克
- 红中麻将
- 转转麻将
- 长沙麻将
- 广东麻将
- 江苏麻将

第一版建议优先选 2-3 个玩法做完整闭环，例如：

- 跑得快
- 240
- 麻将合集中的一种地方麻将

完整玩法库可作为后续阶段扩展。

## 5. 第一版范围

### 5.1 首页 / 大厅

首页要直接服务熟人开局，不做普通游戏大厅堆砌。

核心能力：

- 创建好友房
- 输入房号加入
- 选择常玩玩法
- 查看好友在线
- 查看好友房邀请
- 查看最近牌局或最近战绩
- 查看积分

底部必须有固定位置的一排按钮，作为主导航。

建议底部导航：

- 首页
- 牌友
- 开一桌
- 战绩
- 我的

如果首版需要更简洁，可以先保留：

- 首页
- 好友
- 开一桌
- 战绩
- 我的

### 5.2 创建好友房

创建房间时需要选择：

- 玩法
- 局数
- 人数
- 底分或积分规则
- 特殊规则
- 是否允许观战
- 是否允许好友带人

创建完成后生成：

- 房间号
- 邀请入口
- 微信分享卡片

### 5.3 加入房间

加入方式：

- 输入 6 位房号
- 点击好友邀请
- 从好友房列表进入
- 扫码加入可作为后续能力

### 5.4 好友房状态

好友房需要展示：

- 房主
- 玩法
- 人数，例如 3/4
- 状态，例如等待中、进行中、可旁观
- 底分
- 规则摘要
- 加入按钮

### 5.5 积分体系

第一版积分只做非充值型账户值：

- 每局积分变化
- 最近战绩
- 好友间排名
- 基础等级或称号

不做：

- 充值
- 提现
- 现金兑换
- 付费房卡

## 6. 非目标

第一阶段不做：

- 充值系统
- 复杂商业化
- 强排位赛季
- 大规模赛事
- 复杂俱乐部运营后台
- 多端 App
- 强动画游戏大厅

## 7. 成功标准

第一版成功标准：

- 用户打开后 3 秒内知道如何开一桌
- 创建好友房路径足够短
- 微信分享后好友能快速加入
- UI 第一眼有高级感，不像普通棋牌城
- 底部固定导航清晰
- 积分和战绩能形成轻度留存
