# 项目路线图

更新时间：2026-06-23

## 阶段 0：产品与 UI 定稿

目标：

- 明确第一版产品范围
- 定义首页、创建房间、加入房间、好友邀请、战绩等核心流程
- 确认深色高级 UI 方向
- 形成可用于原型开发的设计约束

产出：

- PRD
- UI 样式要求
- 首页效果图
- 核心流程图

## 阶段 1：微信小程序原型

目标：

- 做可点击的小程序前端原型
- 实现底部固定导航
- 实现首页、创建房间、加入房间、玩法选择、好友邀请、战绩页
- 使用 mock 数据，不接真实对局

建议技术：

- 原生微信小程序，或 Taro / uni-app
- 先以前端原型为主

## 阶段 2：好友房闭环

目标：

- 用户登录
- 创建房间
- 分享房间
- 加入房间
- 房间状态同步
- 基础积分记录

后端能力：

- 用户账户
- 房间服务
- WebSocket 或实时同步
- 战绩与积分存储

## 阶段 3：首批玩法规则

目标：

- 选择 2-3 个玩法先完整实现
- 支持规则配置
- 支持结算
- 支持断线重连

推荐首批玩法：

- 跑得快
- 240
- 一种麻将玩法

## 阶段 4：匹配与扩展

目标：

- 自由匹配
- 好友补位
- 更多玩法
- 排行榜
- 轻度活动

仍然不默认引入充值系统。
